# Week 6: Asynchronous JavaScript

## Overview
This project is part of **Season 10 – Week 6 Asynchronous JavaScript**.  
It demonstrates how to handle asynchronous operations in JavaScript using **callbacks, promises, async/await**, and working with **APIs** to build a weather dashboard.

---

## Features
- Practiced **callbacks, promises, and async/await**  
- Implemented **error handling** with `try...catch`  
- Fetched data from a **weather API**  
- Built a **weather dashboard** that displays current conditions  
- Added **loading states** and error messages for failed requests  

---

## Technologies
- HTML, CSS
- JavaScript (ES6+)
- OpenWeatherMap API
- LocalStorage

## How to Run
1. Open `index.html` in your browser.  
2. Enter a city name in the weather dashboard input.  
3. The app will fetch and display weather data using the API.  

---

## Challenges Faced
- Handling **API errors** when the city name was invalid.  
- Remembering to use `await` only inside `async` functions.  
- Debugging why the dashboard didn’t update until I re‑rendered after fetching data.
